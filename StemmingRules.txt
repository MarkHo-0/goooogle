flow chart

1. pre-processing
Handleing words like "isn't" to prevent losing semantics (especailly for "NOT")

2. Tokenization (most important)
Splitting words by whitespace
Handing words like "mom's", "He's" which would not lose semantics 
other cases, 
unit "1cm" -> 1cm (determined by whitespace)
date "2000/01/02" -> 2000 01 02 (Splitting by forward slash, don't need to determine which one (01 or 02) is date or month)
pure number "1 2 3" -> 1 2 3, "123" -> 123 (determined by whitespace)
abbreviation "HK" -> HK, "hkust" -> hkust, "ABC" -> ABC (We don't need to handle the losing of semantics here. If do, We
need context analysis)
synonyms "spider" -> spider (No need to determine which spider user talk for (animal?crawler?))
moive "Avator2" -> Avator2, "Avator 2" -> Avator 2 (Professional search engines solve this 
with more advanced techniques (like n-grams or phrase indexing) but we don't need)
punctuation "Hi!?" -> Hi, "<h1>" -> h1 (Discarding them)

3. Lowercasing
Avoiding to handle many repeated cases

4. Stemming 
Use library functions to handle suffix:
I.e.
    stemmer.setCurrent(token);
    stemmer.stem();
    stemmedTokens.add(stemmer.getCurrent());
